인공지능과 머신러닝 기초

인공지능(Artificial Intelligence, AI)은 인간의 지능을 모방하여 학습, 추론, 판단 등의 작업을 수행할 수 있는 컴퓨터 시스템을 말합니다. 인공지능은 크게 약인공지능(Narrow AI)과 강인공지능(General AI)으로 구분됩니다.

머신러닝(Machine Learning)은 인공지능의 하위 분야로, 명시적인 프로그래밍 없이 데이터로부터 패턴을 학습하는 알고리즘을 연구합니다. 지도학습, 비지도학습, 강화학습으로 나뉩니다.

딥러닝(Deep Learning)은 머신러닝의 하위 분야로, 인공 신경망(Neural Network)을 여러 층으로 쌓아 복잡한 패턴을 학습합니다. CNN, RNN, Transformer 등의 아키텍처가 대표적입니다.

자연어 처리와 대규모 언어 모델

자연어 처리(Natural Language Processing, NLP)는 컴퓨터가 인간의 언어를 이해하고 생성할 수 있게 하는 기술입니다. 토큰화, 품사 태깅, 개체명 인식, 감정 분석, 기계 번역 등이 주요 태스크입니다.

대규모 언어 모델(Large Language Model, LLM)은 수십억 개 이상의 파라미터를 가진 언어 모델로, 방대한 텍스트 데이터로 사전 학습됩니다. GPT, LLaMA, Qwen, Gemma 등이 대표적인 LLM입니다.

트랜스포머(Transformer)는 2017년 Google이 "Attention is All You Need" 논문에서 제안한 아키텍처입니다. 셀프 어텐션(Self-Attention) 메커니즘을 핵심으로 하며, 입력 시퀀스의 모든 위치를 동시에 처리할 수 있어 병렬화가 가능합니다.

어텐션 메커니즘은 Query, Key, Value 세 가지 벡터를 사용합니다. Query와 Key의 내적(dot product)으로 어텐션 스코어를 계산하고, Softmax를 적용하여 가중치를 구한 뒤, Value에 곱하여 최종 출력을 생성합니다.

멀티헤드 어텐션(Multi-Head Attention)은 여러 개의 어텐션 헤드를 병렬로 수행하여 서로 다른 관점에서 입력을 분석합니다. 이를 통해 모델은 다양한 종류의 관계를 동시에 학습할 수 있습니다.

토큰화와 임베딩

토큰화(Tokenization)는 텍스트를 모델이 처리할 수 있는 작은 단위(토큰)로 분할하는 과정입니다. BPE(Byte Pair Encoding), WordPiece, SentencePiece 등의 알고리즘이 사용됩니다.

BPE(Byte Pair Encoding)는 가장 널리 사용되는 토큰화 알고리즘입니다. 먼저 텍스트를 개별 문자로 분할한 후, 가장 빈번하게 등장하는 문자 쌍을 반복적으로 병합하여 어휘(vocabulary)를 구축합니다.

임베딩(Embedding)은 토큰을 고차원 벡터 공간에 매핑하는 과정입니다. 의미적으로 유사한 토큰은 벡터 공간에서 가까운 위치에 놓이게 됩니다. Word2Vec, GloVe, FastText 등이 전통적인 임베딩 방법이며, 현대 LLM은 트랜스포머 내부에서 문맥을 고려한 임베딩을 학습합니다.

파인튜닝과 효율적 학습

파인튜닝(Fine-tuning)은 사전 학습된 모델을 특정 태스크나 도메인에 맞게 추가 학습하는 과정입니다. 전체 파라미터를 업데이트하는 Full Fine-tuning과 일부만 업데이트하는 Parameter-Efficient Fine-tuning(PEFT)으로 나뉩니다.

LoRA(Low-Rank Adaptation)는 가장 인기 있는 PEFT 방법입니다. 원본 모델의 가중치를 동결(freeze)하고, 작은 랭크의 어댑터 행렬(A, B)만 학습합니다. 전체 파라미터의 1~2%만 학습하면서도 Full Fine-tuning에 근접한 성능을 달성합니다.

QLoRA(Quantized LoRA)는 LoRA에 양자화를 결합한 방법입니다. 모델을 4bit로 양자화하여 메모리를 절약하면서 LoRA 어댑터를 학습합니다. 이를 통해 8GB VRAM의 GPU에서도 7B 모델을 파인튜닝할 수 있습니다.

Continuous Pretraining(지속 사전학습)은 사전 학습된 모델에 도메인 특화 텍스트를 추가로 학습시키는 방법입니다. 의료, 법률, 금융 등 특정 도메인의 지식을 주입하는 데 사용됩니다.

Instruction Tuning(지시 튜닝)은 모델이 사용자의 지시를 잘 따르도록 학습하는 방법입니다. instruction-input-output 형식의 데이터를 사용하며, Alpaca, ShareGPT 등의 데이터 형식이 널리 사용됩니다.

강화학습과 선호도 최적화

RLHF(Reinforcement Learning from Human Feedback)는 인간의 선호도를 반영하여 모델을 개선하는 방법입니다. 보상 모델(Reward Model)을 학습한 후, PPO(Proximal Policy Optimization) 알고리즘으로 정책을 최적화합니다.

DPO(Direct Preference Optimization)는 보상 모델 없이 직접 선호도를 학습하는 방법입니다. chosen(선호 응답)과 rejected(비선호 응답) 쌍을 사용하여 모델을 최적화합니다. PPO보다 구현이 간단하고 안정적입니다.

GRPO(Group Relative Policy Optimization)는 DeepSeek에서 제안한 방법입니다. 하나의 프롬프트에 대해 여러 응답을 생성하고, 그룹 내에서 상대적 보상을 비교하여 학습합니다. Critic 모델이 필요 없어 메모리 효율적입니다.

모델 서빙과 최적화

양자화(Quantization)는 모델의 가중치를 더 낮은 정밀도로 변환하여 메모리와 추론 속도를 개선하는 기술입니다. GPTQ, AWQ, GGUF 등의 양자화 방법이 있으며, 4bit 양자화 시 모델 크기를 약 75% 줄일 수 있습니다.

vLLM은 PagedAttention을 사용하여 높은 처리량의 LLM 서빙을 제공하는 프레임워크입니다. KV 캐시를 효율적으로 관리하여 동시에 많은 요청을 처리할 수 있습니다.

Ollama는 로컬 환경에서 LLM을 쉽게 실행할 수 있게 해주는 도구입니다. 명령줄에서 간단하게 모델을 다운로드하고 실행할 수 있으며, GGUF 포맷의 양자화 모델을 지원합니다.

RAG과 벡터 데이터베이스

RAG(Retrieval-Augmented Generation)는 외부 지식 베이스에서 관련 정보를 검색하여 LLM의 답변을 보강하는 기술입니다. LLM의 환각(hallucination) 문제를 완화하고, 최신 정보를 반영할 수 있습니다.

벡터 데이터베이스는 텍스트를 임베딩 벡터로 변환하여 저장하고, 코사인 유사도 등으로 유사한 문서를 빠르게 검색합니다. ChromaDB, Pinecone, Weaviate, Milvus 등이 대표적입니다.

시맨틱 서치(Semantic Search)는 키워드 매칭이 아닌 의미적 유사도를 기반으로 검색하는 방법입니다. 임베딩 모델로 질문과 문서를 벡터화한 후, 벡터 간 유사도를 계산하여 가장 관련성 높은 문서를 찾습니다.

Advanced RAG 기법으로는 HyDE(Hypothetical Document Embeddings), Reranking, Ensemble Retriever, Parent Document Retriever 등이 있습니다. 이러한 기법들은 기본 RAG의 검색 품질을 향상시킵니다.

평가와 배포

LLM 평가는 자동 메트릭(Perplexity, BLEU, ROUGE, BERTScore)과 인간 평가를 조합하여 수행합니다. 최근에는 LLM-as-a-Judge 방식으로 GPT-4 등의 모델이 다른 모델의 출력을 평가하는 방법이 널리 사용됩니다.

RAGAS는 RAG 시스템의 성능을 평가하는 프레임워크입니다. Faithfulness, Answer Relevancy, Context Precision, Context Recall 등의 메트릭을 제공합니다.

모델 배포 시 양자화, 배칭, KV 캐시 최적화 등을 통해 추론 효율을 높입니다. FastAPI나 Flask로 API 서버를 구축하고, Streamlit으로 웹 인터페이스를 만들 수 있습니다.
